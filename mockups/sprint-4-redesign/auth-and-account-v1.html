What's in v0.10 — Customer auth + My bookings

  • + new   Sign-up modal — same shell as sign-in, full form (name, mobile, email, password+confirm, T&Cs).
  • ↻ overhaul   Sign-in modal — real password sign-in + OTP toggle, replaces the "coming soon" stub.
  • + new   OTP entry step — 6 digit cells, paste support, 30s resend cooldown.
  • + new   Ghost-account claim — when signup mobile already exists from a Concierge booking, prompt OTP to claim.
  • + new   Header signed-in state — name + avatar dropdown with My bookings / Sign out.
  • + new   /my-bookings page — lists customer's concierge inspections with status pill + link to tracker.
  • ↻ enabled   OAuth row (Google active) — Google works; Apple+Facebook stay greyed/"coming soon".
  • + new   Saved cars — heart toggle on every listing card + dedicated /my-saved-cars page. Requires sign-in (prompts sign-in modal if guest clicks heart).
  • + new   Edge states — locked account, OTP expired, weak password, mobile already in use.

Modals are shown INLINE for review (not actually overlay-positioned). In production they appear as centered overlays over the storefront.
