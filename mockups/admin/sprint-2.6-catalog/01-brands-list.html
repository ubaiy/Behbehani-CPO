Role gating The "+ Add brand" CTA, edit pencil column, and status toggle are wrapped in *adminRole="['content_editor','general_manager','super_admin']". Non-write roles see the table read-only — buttons removed from the DOM, not disabled. Today's mockup is the super_admin view.
Soft delete only The status column is a single Active/Inactive toggle — no Delete button anywhere in the module. Inactive brands disappear from customer browse + the create-listing dropdown, but existing listings that reference them keep displaying (Mitsubishi row demonstrates: 5 models, 23 active listings).
Confirm modal on deactivate Toggling Active → Inactive opens the shared ConfirmModalService in destructive variant:
"Deactivating Toyota will remove it from customer browse and the create-listing dropdown. 47 existing listings will continue to display correctly. Confirm?"
Logo column Logos use the 36×36 rounded tile. Placeholder shows brand initial in brand color. Real logoUrl renders as <img> with object-fit: contain. Missing logo → initial fallback (Toyota T, Lexus L, etc.).
