Confirmation Modal

Used for all destructive or irreversible admin actions. Replaces window.confirm and follows the existing StageTransitionModal styling. Implemented with the native <dialog> element — no @angular/animations dependency required.

Variant decision guide

Standard — variant: 'confirm'

Reversible or low-risk actions. Stage transitions, revert to draft, unpublish. Uses brand-600 confirm button.

Destructive — variant: 'destructive'

High-impact but restorable actions. Archive, bulk remove, suspend. Optional requireTyped string. Uses red-600 confirm button.

Delete — variant: 'delete'

Permanent, unrecoverable deletions. Pricing tiers with active listings, user records. Uses red-700 confirm button.

Behavior & Interaction

Trigger Behavior
TAB Cycles focus between Cancel and Confirm only. Focus is trapped inside the modal — page content behind is unreachable until modal closes.
ESC Resolves the promise with 'cancel'. Equivalent to clicking Cancel. Native <dialog> fires cancel event automatically.
ENTER Activates whichever button currently holds focus. Initial focus is set on Cancel to prevent accidental confirmation on rapid keyboard use.
Backdrop click Click outside the modal card resolves with 'cancel'. Detected via pointerdown on <dialog> itself (not on the inner card).
After confirm Confirm button label is replaced by a spinner ( loading) until the promise resolves or rejects. Both buttons are disabled during this state.
Telemetry Every confirmed action fires an audit log entry: admin_action: confirm_<key> where key is the caller-supplied action key (e.g. archive_vehicle, delete_pricing_tier).

Loading state — standard (button disabled)

Loading state — destructive (button disabled)

Typed confirmation — disabled state (value empty)

Confirm button enabled only when input value matches the required string exactly (case-sensitive).

Usage Examples

Illustrative API signatures — not compiled Angular code
Variant A — Standard confirm (stage transition / revert to draft) 
// Inject the service, call .open(), await the result
const result = await this.confirm.open({
  title:        'Revert listing to draft?',
  body:         'This will remove the listing from public view. You can publish it again afterward.',
  confirmLabel: 'Revert to draft',
  variant:      'confirm',          // brand-600 confirm button, info icon
  actionKey:    'revert_to_draft',  // used for audit log: admin_action: confirm_revert_to_draft
});

if (result === 'confirm') {
  await this.listingService.revertToDraft(listingId);
}
Variant B — Destructive confirm with typed confirmation (archive vehicle) 
// requireTyped forces the user to type the string before confirming
const result = await this.confirm.open({
  title:         `Archive ${vehicle.year} ${vehicle.make} ${vehicle.model}?`,
  subtitle:      `VIN ••••••${vehicle.vinSuffix}`,  // rendered below title in small mono text
  body:          'Archived vehicles are removed from inventory listings and the storefront.'
               + ' Audit history is preserved. You can restore from the Archive view.',
  warningNote:   'This action is logged in the audit trail.',
  confirmLabel:  'Archive vehicle',
  variant:       'destructive',     // red-600 confirm button
  requireTyped:  'ARCHIVE',         // confirm button disabled until user types this exactly
  actionKey:     'archive_vehicle',
});

if (result === 'confirm') {
  await this.vehicleService.archive(vehicle.id);
}
Variant C — Delete severe (pricing tier with active listings) 
// 'delete' variant renders a red top stripe and uses red-700 button
const result = await this.confirm.open({
  title:        `Delete '${tier.name}' pricing tier?`,
  body:         `This tier is currently applied to ${tier.listingCount} listings.`
              + ' Deleting it will not roll back discounts already applied,'
              + ' but no future runs will use this rule.',
  callout:      'This action cannot be undone.',  // renders triangle-warn callout in red-50
  confirmLabel: 'Delete tier',
  variant:      'delete',              // red-700 button + red accent stripe at top
  actionKey:    'delete_pricing_tier',
});

if (result === 'confirm') {
  await this.pricingService.deleteTier(tier.id);
}
