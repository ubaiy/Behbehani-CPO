Behavior summary (locked decisions)

  • Single rounded card replaces today's 3 stacked cards (Gallery + Walk-around Video + 360° Spin).
  • Desktop (≥ sm): tab strip ABOVE the 16:10 viewer. Mobile (< sm): tab strip BELOW the viewer so hero image lands above the fold.
  • Tab strip auto-hides when listing has only one media type (e.g. Photos only).
  • Each tab's body reuses the same 16:10 aspect box → no layout shift on swap.
  • Video state: switching away pauses; switching back resumes from last currentTime. Video element stays mounted in the DOM.
  • 360° drag-to-scrub overlay is identical to today's implementation — just hosted inside the unified card.
  • Thumb strip is shown only on the Photos tab.
  • "View All" + Fullscreen buttons live at the top-right of the tab strip (desktop). On mobile they collapse into floating overlay buttons on the viewer.
  • RTL: tab order reverses, prev/next arrows mirror (kept as today).
  • SSR: tab labels + Photos panel render on server; Video + 360° lazy-load via IntersectionObserver only when their tab first becomes active.
  • i18n keys to add: vdp.media.tabs.photos · vdp.media.tabs.video · vdp.media.tabs.spin · vdp.media.viewAll. Existing vdp.walkaround.* + vdp.spin360.* keys are repurposed under the new component.
