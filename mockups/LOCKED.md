# mockups/LOCKED.md — approved mockup registry

> When a mockup is approved for implementation, append one row here. Implementation agents read this BEFORE touching code. After implementation, the row gets a `[SHIPPED]` marker with the Angular/RN target.

## Schema
| Date | Mockup path | Owner | Status | Target |
|---|---|---|---|---|
| YYYY-MM-DD | mockups/path/file.html | A/B/C | locked / shipped / superseded | apps/web/src/app/features/... or N/A |

**Owner key**
- **A** — storefront / web / account paths (`sprint-*-web`, `sprint-*-account`, `sprint-*-redesign`, `web/`, `sprint-5-*`)
- **B** — admin / CPO paths (`admin/`, `sprint-*-admin`, `sprint-*-cpo`, `sprint-*-users`, `sprint-*-catalog`, `sprint-*-inspection`, `phase-4-offer/admin-*`)
- **C** — mobile / React-Native paths (`mobile/sprint-M*`)
- **AB** — shared between admin + storefront (phase-4 offer pages that have both admin + customer variants)
